This guide covers installing dependencies, configuring environment variables, and running the development server for the Crepes & Waffles Coworking platform.

## Prerequisites

Before you begin, ensure you have:

- **Node.js** 18+ and npm 9+ installed
- Access to the employee API (`https://apialohav2.crepesywaffles.com`)
- Access to the reservations API (`https://macfer.crepesywaffles.com`)
- (Optional) WebSocket server URL for real-time sync

<Note>
The platform is built with **React 19**, **Vite 7**, **React Router 7**, and **Ant Design 6**. See `package.json` for the full dependency list.
</Note>

## Environment Variables

The platform uses environment variables for configuration. Create a `.env` file in the project root.

### Required Variables

<Note>
This platform doesn't require any mandatory environment variables to run locally. The API endpoints are hardcoded in the source code. However, you can configure optional variables for enhanced features.
</Note>

### Optional Variables

#### WebSocket Configuration

Enable real-time synchronization so multiple users see reservation updates instantly without reloading.

```bash .env
# WebSocket endpoint for real-time reservation updates
VITE_RESERVAS_WS_URL=wss://tu-servidor/ws/reservas

# Polling interval in milliseconds (fallback when WebSocket unavailable)
VITE_RESERVAS_POLLING_MS=30000
```

**How it works:**

- If `VITE_RESERVAS_WS_URL` is set, the app connects to WebSocket for instant updates
- If WebSocket fails or isn't configured, the app polls the API every `VITE_RESERVAS_POLLING_MS` milliseconds (default: 30 seconds)
- Local events still work without WebSocket — users see their own changes immediately

<CodeGroup>

```javascript With WebSocket
// src/hooks/useRealtimeSync.js
const ws = new WebSocket(import.meta.env.VITE_RESERVAS_WS_URL);

ws.onmessage = (event) => {
  if (event.data === 'reservas-updated') {
    refreshCallback(); // Reload reservations
  }
};
```

```javascript Without WebSocket (Polling)
// Fallback to polling every 30 seconds
const interval = setInterval(() => {
  refreshCallback();
}, import.meta.env.VITE_RESERVAS_POLLING_MS || 30000);
```

</CodeGroup>

<Warning>
WebSocket URLs must use the `wss://` protocol (secure WebSocket). Using `ws://` will fail in production environments with HTTPS.
</Warning>

### Environment Variable Access

Vite exposes environment variables prefixed with `VITE_` to your client-side code:

```javascript
// Access in your components
const wsUrl = import.meta.env.VITE_RESERVAS_WS_URL;
const pollingMs = import.meta.env.VITE_RESERVAS_POLLING_MS;
```

<Tip>
Never commit your `.env` file to version control. Add it to `.gitignore` to protect sensitive configuration.
</Tip>

## Deployment

Deploy the platform to production in three steps:

<Steps>
  <Step title="Build the application">
    ```bash
    npm run build
    ```

    This generates the `dist/` folder with production-ready files.
  </Step>

  <Step title="Configure environment variables">
    Set production environment variables in your hosting platform:

    ```bash
    VITE_RESERVAS_WS_URL=wss://your-production-ws-server/ws/reservas
    VITE_RESERVAS_POLLING_MS=30000
    ```

    <Note>
    Environment variables must be set **before** building. Vite bakes them into the bundle at build time.
    </Note>
  </Step>

  <Step title="Deploy to hosting">
    Upload the `dist/` folder to your hosting service.

    <CodeGroup>

    ```bash Vercel
    npm install -g vercel
    vercel --prod
    ```

    ```bash Netlify
    npm install -g netlify-cli
    netlify deploy --prod --dir=dist
    ```

    ```bash AWS S3 + CloudFront
    aws s3 sync dist/ s3://your-bucket-name
    aws cloudfront create-invalidation --distribution-id YOUR_ID --paths "/*"
    ```

    ```bash Docker
    # Dockerfile
    FROM nginx:alpine
    COPY dist/ /usr/share/nginx/html
    EXPOSE 80
    CMD ["nginx", "-g", "daemon off;"]

    # Build and run
    docker build -t coworking-app .
    docker run -p 80:80 coworking-app
    ```

    </CodeGroup>
  </Step>
</Steps>

### SPA Routing Configuration

The app uses React Router for client-side routing. Configure your server to redirect all requests to `index.html`:

<CodeGroup>

```nginx Nginx
server {
  listen 80;
  server_name example.com;
  root /var/www/dist;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;
  }
}
```

```apache Apache (.htaccess)
RewriteEngine On
RewriteBase /
RewriteRule ^index\.html$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]
```

```json Vercel (vercel.json)
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/" }
  ]
}
```

```toml Netlify (netlify.toml)
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

</CodeGroup>

<Warning>
Without proper SPA routing configuration, direct navigation to routes like `/reservas` or `/panel` will result in 404 errors.
</Warning>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="npm install fails with peer dependency errors">
    **Problem:** React 19 is still relatively new and some packages may show peer dependency warnings.

    **Solution:** Use the `--legacy-peer-deps` flag:
    ```bash
    npm install --legacy-peer-deps
    ```

    Or add this to your `.npmrc`:
    ```bash
    legacy-peer-deps=true
    ```
  </Accordion>

  <Accordion title="Port 5173 is already in use">
    **Problem:** Another process is using the default Vite port.

    **Solution:** Specify a different port:
    ```bash
    npm run dev -- --port 3000
    ```

    Or update `vite.config.js`:
    ```javascript
    export default defineConfig({
      plugins: [react()],
      server: { port: 3000 },
    })
    ```
  </Accordion>

  <Accordion title="API requests fail with CORS errors">
    **Problem:** Browser blocks requests to external APIs due to CORS policy.

    **Solution:** The backend APIs must include proper CORS headers. If you control the backend, add:
    ```javascript
    Access-Control-Allow-Origin: *
    Access-Control-Allow-Methods: GET, POST, PUT, DELETE
    Access-Control-Allow-Headers: Content-Type
    ```

    Alternatively, use Vite's proxy in development:
    ```javascript vite.config.js
    server: {
      proxy: {
        '/api': 'https://macfer.crepesywaffles.com',
      },
    }
    ```
  </Accordion>

  <Accordion title="Environment variables not working">
    **Problem:** Variables don't load or show as `undefined`.

    **Solutions:**
    1. Ensure variables are prefixed with `VITE_`
    2. Restart the dev server after changing `.env`
    3. Access with `import.meta.env.VITE_VAR_NAME` (not `process.env`)
    4. Variables are embedded at build time — rebuild after changes
  </Accordion>

  <Accordion title="WebSocket connection fails">
    **Problem:** Real-time sync doesn't work.

    **Solutions:**
    1. Check `VITE_RESERVAS_WS_URL` uses `wss://` (not `ws://`)
    2. Verify WebSocket server is running and accessible
    3. Check browser console for connection errors
    4. The app will fall back to polling automatically

    Test WebSocket manually:
    ```javascript
    const ws = new WebSocket('wss://your-server/ws/reservas');
    ws.onopen = () => console.log('Connected!');
    ws.onerror = (err) => console.error('Error:', err);
    ```
  </Accordion>
</AccordionGroup>
