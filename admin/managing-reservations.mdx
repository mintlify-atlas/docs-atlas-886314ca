## Overview

Administrators have enhanced reservation management capabilities that bypass standard user restrictions. This includes manual confirmation without location or time constraints, and the ability to reactivate canceled reservations.

## Admin Tabs

The admin panel provides three filtering tabs for viewing reservations:

```javascript
{
  { key: 'todos',  label: 'Todas' },
  { key: 'mis',    label: 'Mis reservas' },
  { key: 'otros',  label: 'Otras' },
}
```

- **Todas** - Shows all reservations from all users
- **Mis reservas** - Shows only the admin's own reservations
- **Otras** - Shows reservations created by other users

## Manual Confirmation

### How It Works

Admins can confirm any pending reservation without the standard restrictions that apply to regular users:

```javascript
if (esAdmin) {
  setConfirmando(id);
  try {
    const mensaje = 'Reserva confirmada por administrador.';

    await updateReservaWithVerification(id, {
      estado: 'Confirmada',
      confirmada: true,
      verificacionAsistencia: {
        fecha: new Date().toISOString(),
        mensaje,
        tipo: 'confirmacion-admin-manual',
        autorizadaPor: documentoUsuario,
        nombreAutorizador: datosEmpleado?.nombre ?? 'Administrador',
      },
    }, reservaAux);
    
    // ... state update
    alert('Reserva confirmada por administrador.');
  } catch (err) {
    console.error(err);
    alert(err?.message || 'Error al confirmar la reserva.');
  } finally {
    setConfirmando(null);
  }
  return;
}
```

**Source:** `Panel.jsx:557-602`

### Verification Record

When an admin confirms a reservation, the system records:

- **tipo**: `'confirmacion-admin-manual'`
- **autorizadaPor**: Admin's document number
- **nombreAutorizador**: Admin's name
- **fecha**: Timestamp of confirmation
- **mensaje**: `'Reserva confirmada por administrador.'`

### User Experience

For admins, pending reservations show a helper message:

```javascript
helperMessage={esAdmin && r.estado === 'Pendiente'
  ? (r.documento === documentoUsuario
      ? 'Puedes confirmar esta reserva como administrador.'
      : 'Puedes confirmar manualmente esta reserva como administrador.')
  : ''}
```

## Reactivating Canceled Reservations

### Reactivation Function

Only admins can reactivate canceled reservations:

```javascript
const handleReactivar = async (id) => {
  if (!esAdmin) return;

  const reservaAux = reservations.find(r => r.id === id);
  if (!reservaAux || reservaAux.estado !== 'Cancelada') {
    return;
  }

  setReactivando(id);
  try {
    await updateReservaWithVerification(id, {
      estado: 'Pendiente',
      confirmada: null,
      motivoCancelacion: null,
      motivo_cancelacion: null,
      verificacionAsistencia: {
        fecha: new Date().toISOString(),
        mensaje: 'Reserva reactivada por administrador.',
        tipo: 'reactivacion-admin',
        autorizadaPor: documentoUsuario,
        nombreAutorizador: datosEmpleado?.nombre ?? 'Administrador',
      },
    }, reservaAux);

    // ... state update
    alert('Reserva reactivada.');
  } catch (err) {
    console.error(err);
    alert(err?.message || 'Error al reactivar la reserva.');
  } finally {
    setReactivando(null);
  }
};
```

**Source:** `Panel.jsx:674-726`

## Search and Filtering

### Admin Search Capabilities

Admins have enhanced search functionality:

```javascript
placeholder={esAdmin ? "Buscar por ID, nombre, cédula…" : "Buscar por ID de reserva…"}
```

**Source:** `Panel.jsx:1103`

Search filters include:
- Reservation ID
- User's full name (`nombreCompleto`)
- User's short name (`nombre`)
- Document number (`documento`)
- Desk ID (`puestoId`)

```javascript
if (searchQuery.trim()) {
  const q = searchQuery.trim().toLowerCase();
  result = result.filter(r =>
    String(r.id).includes(q) ||
    (r.nombreCompleto ?? '').toLowerCase().includes(q) ||
    (r.nombre ?? '').toLowerCase().includes(q) ||
    (r.documento ?? '').toLowerCase().includes(q) ||
    String(r.puestoId ?? '').includes(q)
  );
}
```

**Source:** `Panel.jsx:816-825`

### Filter Options

Admins can combine multiple filters:

1. **Estado (Status)**: Todos, Pendiente, Confirmada, Cancelada
2. **Sala (Room)**: Filter by specific rooms
3. **Admin Tabs**: Todas, Mis reservas, Otras
4. **Search Query**: Free text search
